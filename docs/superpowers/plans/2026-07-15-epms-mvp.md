# EPMS MVP Implementation Plan

> **For agentic workers:** Implement inline in the current workspace. Keep each calculation change test-first and do not introduce future modules.

**Goal:** Deliver the approved Engineering-owned EPMS vertical slice.

**Architecture:** Next.js App Router, Better Auth, PostgreSQL, server-only DAL, Server Actions, and pure tested engineering calculations.

**Tech Stack:** Next.js 16.2.10, React 19.2.4, TypeScript, PostgreSQL, Better Auth, Tailwind CSS 4.

## Global Constraints

- Time is stored and calculated in seconds.
- Approved Standard Times and completed scenarios are immutable.
- Server Actions recheck authentication and authorization.
- The UI follows `emil-design-eng`: no high-frequency navigation animation, exact transition properties, press feedback, and reduced-motion support.
- No Production actuals, OEE, Pareto, incident workflow, or reusable importer.
